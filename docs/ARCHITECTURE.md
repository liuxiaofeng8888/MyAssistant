## 架构设计（Android Java + 服务端）

### 1. 总体架构
#### 1.1 模块划分
- **Android 客户端（Java）**
  - UI/会话：对话页、会话列表、设置
  - 录音：采集 PCM/OPUS（建议 OPUS）
  - 播报：Android System TTS（可打断）
  - 本地存储：会话/消息/设置（Room 或 SQLite）
  - 网络：WebSocket（流式）或 HTTP（非流式）
- **服务端（推荐做“语音编排层 Orchestrator”）**
  - 会话管理：session、上下文裁剪
  - ASR 接入：流式 ASR（第三方）或自建
  - LLM/NLU：意图识别 + 参数提取 + 回复生成
  - 工具/技能执行：提醒/待办/天气/计时器等
  - 安全与治理：鉴权、限流、审计、日志脱敏
  - 可观测：指标、追踪、告警

#### 1.2 数据流（建议：流式）
1. 客户端开始录音 → 通过 **WebSocket** 发送音频分片
2. 服务端流式 ASR → 返回 partial/final 文本
3. 服务端触发 LLM：识别是“聊天”还是“工具调用”
4. 若工具调用：执行技能 → 得到结构化结果
5. 服务端流式返回 assistant 文本（可边生成边返回）
6. 客户端展示并 TTS 播报（支持中途停止/打断）

### 2. 关键设计决策（MVP）
- **客户端不直连第三方**：所有第三方密钥放服务端，便于替换与控成本。
- **优先系统 TTS**：端侧播报省成本、延迟低；二期再引入云 TTS 音色。
- **统一工具协议**：LLM 工具调用输出严格 JSON（服务端校验），避免“幻觉执行”。

### 3. 会话与上下文策略
- **会话粒度**：每次进入对话页创建/选择一个 `conversation_id`。
- **上下文裁剪**：保留最近 N 条消息 + 关键“记忆”（可选），避免 token 失控。
- **可控开关**：设置中支持“禁用上下文（每次单轮）”。

### 4. 错误处理与降级
- **ASR 失败**：提示“没听清/网络问题”，提供重试。
- **LLM 失败**：返回固定模板 + 建议重试/改用文本输入。
- **工具失败**：明确说明失败原因（如“天气服务不可用”），并返回可重试。
- **超时**：
  - ASR 流：超过阈值自动结束并处理已有文本
  - 工具调用：超过阈值返回超时并记录 trace_id

### 5. 安全
- **鉴权**：MVP 可用匿名设备 token；上线建议账号体系或短期 token。
- **限流**：按设备/用户/IP 维度，区分 ASR 与 LLM 配额。
- **审计**：记录工具调用（不记录原始音频，或需显式开关）。
- **日志脱敏**：手机号/地址等字段脱敏。

### 6. 可观测性（建议指标）
- **时延拆分**：upload、asr、llm、tool、first_token、total
- **成功率**：asr_success、tool_success、overall_success
- **成本**：asr_seconds、llm_tokens、tool_qps
