## 语音助手 APP（Android Java + 服务端）PRD（MVP）

### 1. 背景与目标
- **产品目标**：做一个可对话、可执行任务、可扩展技能的语音助手 APP，优先保证“好用、快、稳定、可控（隐私）”。
- **MVP 目标**：打通端到端闭环，并交付 3–5 个高频技能（提醒/待办/天气/计时器/文本整理其一）。
- **成功指标（MVP）**：
  - **端到端成功率**：\(\ge 95\%\)（能给出可用答复/或明确失败提示）
  - **技能执行成功率**：\(\ge 90\%\)
  - **P95 首字响应**：\(\le 2s\)（含 ASR 首段 + 服务端首包）
  - **P95 完整响应**：\(\le 8s\)（视 LLM/网络可调）

### 2. 范围（Scope）
#### 2.1 In Scope（MVP）
- **输入**：点击/按住说话（先不做唤醒词），支持打断。
- **ASR**：一句话或流式识别（建议流式以提升体验），显示识别文本。
- **对话**：支持上下文（同一会话内），支持“闲聊 + 任务指令”。
- **技能框架**：统一的“意图识别/参数提取/执行/回读”。
- **TTS**：播报回答，支持暂停/停止。
- **历史记录**：会话列表、消息详情、删除、复制。
- **设置**：TTS 音色/语速（若系统支持）、是否保存历史、清除数据。
- **隐私**：权限说明、数据收集说明、一键清除。

#### 2.2 Out of Scope（MVP 不做）
- **常驻后台唤醒词**（二期，涉及平台限制与功耗）
- **离线 ASR/TTS/LLM**（二期）
- **插件市场/第三方技能生态**（二期）
- **多语言大规模适配**（二期）

### 3. 目标用户与场景
- **目标用户**：需要免手操作与轻量效率的人群。
- **核心场景**：
  - **提醒/计时**：做饭、喝水、会议前提醒
  - **待办**：口述新增、完成、查询
  - **天气**：询问今日/未来天气
  - **文本整理（可选）**：把口述转成结构化笔记

### 4. 核心用户故事（User Stories）
- 用户点击麦克风说：“提醒我 30 分钟后喝水”，助手创建提醒并播报确认。
- 用户说：“把刚才的提醒取消”，助手识别目标并取消，播报结果。
- 用户说：“明天北京天气怎么样”，助手查询并播报。
- 用户在弱网/失败时，能收到明确提示与可重试按钮。

### 5. 交互流程（MVP）
1. 用户点击麦克风开始录音（UI：波形/音量条 + “正在聆听…”）
2. ASR 输出文本（可显示中间结果）
3. 服务端生成“意图 + 参数 + 回答草案/工具调用”
4. 如需工具调用：服务端执行技能并生成最终答复
5. 客户端展示答复并 TTS 播报（支持打断）
6. 写入历史（文本为主；音频默认不保存）

### 6. MVP 技能列表（建议）
#### 6.1 提醒/闹钟（P0）
- 新建提醒：时间点/相对时间/重复规则（MVP 可仅做相对时间+时间点）
- 查询提醒：今天/最近
- 取消提醒：按最近一次或按标题匹配（需要二次确认的场景）

#### 6.2 待办（P0）
- 新增待办：标题、可选截止时间
- 完成待办：按最近/按标题匹配
- 列出待办：未完成列表

#### 6.3 天气（P0）
- 当前天气/今日/未来 3 天
- 城市：定位（若授权）或语音中城市名

#### 6.4 计时器（P0）
- 开始/停止/剩余时间

#### 6.5 文本整理（P1，可选）
- “把这段话整理成要点/会议纪要/待办清单”

### 7. 非功能需求（NFR）
- **可靠性**：ASR/LLM/工具任一失败都要降级并提示原因（可重试）。
- **可观测性**：端到端耗时拆分（录音/ASR/LLM/工具/TTS）。
- **隐私**：默认不上传保存原始音频；如需调试采样，必须显式开关与可删除。
- **安全**：接口鉴权、速率限制、敏感信息脱敏日志。

### 8. 约束与假设（MVP）
- Android 端使用系统录音 API + 音频流上传服务端。
- TTS 优先使用 Android 系统 TTS（降低成本与复杂度），服务端仅返回文本（可扩展为云 TTS）。
- 服务端负责对接第三方 ASR/LLM/天气等，客户端不直连第三方（便于换供应商与控钥匙）。

### 9. 验收标准（MVP）
- 能完成：提醒/待办/天气/计时器四项技能的新增与查询/停止等核心路径。
- 断网/超时/权限拒绝时：UI 有清晰提示，且不会崩溃。
- 历史可查看、可删除；设置中可一键清除所有本地数据。
