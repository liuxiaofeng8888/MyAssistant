## 测试计划（MVP）

### 1. 测试范围
- 语音输入/录音权限
- WebSocket 连接与断线重连
- ASR 中间/最终结果展示
- 对话回复展示与 TTS 播报（含打断）
- 技能：提醒/待办/天气/计时器
- 历史记录与设置
- 异常与降级（弱网、超时、第三方失败）

### 2. 关键用例（高优先级）
#### 2.1 权限与基础链路
- 首次启动拒绝麦克风权限：给出说明与去设置入口；不崩溃
- 允许权限后：可正常录音并发送
- 录音中点击停止：能收到完整回复
- 录音中点击取消：服务端停止处理（或客户端停止播放/展示）

#### 2.2 TTS
- 回复开始播报后，用户再次按住说话：**立即停止播报**并开始新一轮
- TTS 开关关闭：只展示文本不播报

#### 2.3 技能：提醒
- “提醒我 30 分钟后喝水” → 成功创建 → 回读时间点
- “取消刚才的提醒” → 取消成功（如存在歧义需二次确认）
- 无法解析时间：“提醒我一会儿喝水” → 追问澄清

#### 2.4 技能：待办
- “添加待办：周五前交报告” → 新增成功
- “完成待办：交报告” → 标记完成（歧义时追问）
- “列出未完成待办” → 返回列表

#### 2.5 技能：天气
- 未授权定位：“北京今天什么天气” → 返回北京
- 授权定位：“今天什么天气” → 返回当前位置

#### 2.6 技能：计时器
- “计时 10 分钟” → 开始 → 通知/提示
- “停止计时” → 停止成功

### 3. 异常与弱网
- WebSocket 建立失败：提示可重试；不阻塞 UI
- 处理中网络断开：提示“网络中断”，允许重试
- ASR/LLM/工具超时：展示超时提示 + 重试按钮

### 4. 性能与稳定性
- 统计 P50/P95：ASR 首段、LLM 首包、总耗时
- 长会话（>30 轮）上下文裁剪是否生效
- 连续发言 10 次是否有内存泄漏/音频资源未释放
