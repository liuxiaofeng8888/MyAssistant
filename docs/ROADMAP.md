## 里程碑与排期（建议）

### 第 0 阶段：对齐与准备（2–3 天）
- PRD（MVP）定稿：范围、技能清单、验收标准
- 技术选型：ASR/LLM/天气服务、WebSocket 协议定稿
- 隐私政策草案：权限、数据收集、保存与删除

### 第 1 阶段：端到端闭环（1–2 周）
- Android（Java）
  - 录音采集 + WebSocket 上传
  - 显示 ASR 文本、显示 assistant 文本
  - 系统 TTS 播报与打断
  - 本地会话与消息存储
- 服务端
  - WebSocket 网关
  - ASR 接入（流式）
  - LLM 接入（先只做对话，不做工具）
  - 基础鉴权、限流、trace_id

### 第 2 阶段：技能落地（1–2 周）
- reminder/todo/weather/timer 四个技能
- 工具调用 schema 校验 + 失败码规范
- 埋点与监控：成功率、耗时、成本

### 第 3 阶段：体验与稳定性（1 周）
- 弱网/超时降级
- UI 细节：录音动画、加载态、重试
- 崩溃与泄漏修复

### 第 4 阶段：灰度与发布（3–5 天）
- 内测/灰度：收集日志与反馈
- 隐私政策与上架材料准备
