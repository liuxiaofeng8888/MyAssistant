## 隐私与安全（MVP）

### 1. 权限清单（Android）
- **麦克风**：用于语音输入（必须）
- **网络**：用于连接服务端（必须）
- **通知**：用于提醒/计时器通知（建议）
- **定位（可选）**：用于天气自动定位（不授权则让用户说城市）

### 2. 数据收集与使用（建议口径）
- **语音音频**：用于实时语音识别与生成回复；默认不在服务端长期保存音频。
- **识别文本与对话文本**：用于完成对话与技能执行；默认保存在本地历史（可关闭/可清除）。
- **诊断日志**：仅包含错误码、耗时、trace_id 等，不包含可识别个人内容（或做脱敏）。

### 3. 用户控制
- **关闭历史记录**：关闭后不保存本地会话内容（或仅保存最近一条用于显示）。
- **一键清除**：清除本地所有会话/设置缓存。
- **导出（可选）**：导出会话为文本（MVP 可后置）。

### 4. 服务端安全措施（最低要求）
- **鉴权**：token 校验；token 与设备/用户绑定。
- **限流与配额**：防刷与控成本（ASR 秒数、LLM tokens）。
- **输入校验**：工具调用参数必须 schema 校验，拒绝越权字段。
- **日志脱敏**：手机号、地址、身份证等模式脱敏。
- **密钥管理**：第三方 API key 不下发客户端；服务端环境变量/密钥库管理。

### 5. 合规注意点（MVP）
- 在应用内提供 **《隐私政策》** 与 **《用户协议》**（至少包含音频上传、用途、保存策略、第三方共享说明）。
- 若未来引入“用于模型训练/质检”的数据采样，必须做 **显式开关** 与 **可撤回**。
